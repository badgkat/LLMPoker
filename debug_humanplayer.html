Debug Human Player Data Flow

Instructions:

  1. Open Chrome DevTools (F12)
  2. Go to Console tab
  3. Navigate to http://localhost:3000
  4. Enter your name in the input field
  5. Click "Start Tournament"
  6. Watch the console for detailed logs

What to Look For:

  • SetupScreen logs: Should show playerSetup object with humanPlayer.name
  • Input onChange: Should log when typing in the name field
  • GameEngine logs: Should show seat assignment process
  • Player creation: Should show human player object with assigned seat
  • Validation errors: Should show clear error messages if something fails

Expected Flow:

1. Initial playerSetup: { humanPlayer: { name: 'You' }, aiPlayers: [...] }
2. User types name: onChange logs each keystroke
3. Updated playerSetup: { humanPlayer: { name: 'UserName' }, aiPlayers: [...] }
4. Click Start Tournament
5. Validation passes
6. Seat assignment: shuffledSeats = [3,1,7,2,0,8,4,5,6] (example)
7. Human player created: { id: 0, name: 'UserName', seat: 3, ... }
8. AI players created with remaining seats
9. Game starts successfully

If you see issues: Check the console for specific error messages. The new validation should provide clear feedback about what's wrong.
